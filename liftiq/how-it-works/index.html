Founding-partner pilot workflow

LiftIQ is a browser-based dispatcher pilot workflow for crane and lifting operations.

Pressure Systems is developing LiftIQ as a pilot-stage dispatch intelligence layer for operators who need visible reasoning before a dispatcher, planner, or branch lead commits the job. The live pilot workflow now covers worker onboarding, checks before allocation, explainable ranking, override capture, and audit-backed proof.

Current logic path

What LiftIQ does now, in truthful terms.

LiftIQ now supports a browser-based dispatcher pilot workflow: import or create workers, record credentials and fatigue, create jobs, run explainable SmartRank recommendations, review hard blocks and warnings, confirm allocations, capture override reasons where needed, and keep allocation-level audit proof.

1

Import or create workers

Dispatch teams can onboard workers from CSV or pasted spreadsheet data, then add or tidy records manually when needed.

2

Record credentials and fatigue

Worker credentials and fatigue records are entered before job allocation so credential gaps, fatigue warnings, and hard blocks are visible early.

3

Create jobs and run SmartRank

Jobs carry the role, crane class, credentials, and task context needed to rank workers, show score breakdowns, and expose warning or hard-block states clearly.

4

Review warnings and confirm allocation

Dispatchers remain the decision-maker, review CredentialGate and FatigueGuard results, and provide an override reason for warning or lower-ranked selections when the workflow requires it.

5

Write audit events and metrics

Confirmed allocations create append-only audit events, update pilot metrics, and keep an operational proof trail inside the pilot workflow.

6

Learn from dispatcher-approved patterns

Manual, imported, and learned preference signals can influence future SmartRank results, but hard blocks still override preference signals every time.

Current pilot truth

Live founding-partner pilot console with bounded workflow scope.

  • Worker import from CSV or pasted spreadsheet data plus manual worker entry.
  • Credential and fatigue checks before allocation.
  • Explainable SmartRank recommendations with visible score breakdowns.
  • Append-only allocation-level audit events and pilot metrics within pilot scope.

Boundaries still in place

Dispatcher-led workflow, not autonomous allocation.

  • Dispatcher remains the decision-maker and final confirmer.
  • No payroll, invoicing, or FCC-replacement claim.
  • No WHS or legal-compliance guarantee claim.
  • Any Xero or MYOB alignment remains later, access-dependent work only.
Workflow fit

LiftIQ sits beside dispatch, workforce, and planning workflows.

It is not being framed as an isolated scoring toy. The pilot workflow connects worker data, job context, visible checks, allocation confirmation, and audit proof while keeping maturity claims disciplined.

Dispatch Import workers, create jobs, rank crew-fit options, review warnings, and confirm allocations with visible reasoning support.
Workforce Credential visibility, fatigue exposure, repeated over-allocation awareness, and preference signals that remain explainable.
Audit Append-only allocation-level audit events and pilot metrics exist inside the pilot workflow.
Finance Planned later alignment with systems such as Xero and MYOB for job-cost and labour-cost context, not current support.
Truth boundary: LiftIQ does not claim autonomous dispatch, WHS/legal compliance guarantees, payroll logic, invoice automation, or live Xero or MYOB integration on this page. Those remain later architecture questions only if pilot evidence and client system access justify them.
