# DispatchTalon Brand Guide

This is the locked DispatchTalon brand mark system.

DispatchTalon uses the approved brand board supplied by Cody as the source of truth. Do not redesign, reinterpret, recolour, stretch, or replace this mark with earlier generated concepts.

## Colour Palette

| Colour | Hex | Use |
|---|---|---|
| Charcoal | `#0E1216` | Primary background and dark UI field. |
| Cream | `#F2F0E6` | Main mark, primary text, high-contrast lockup. |
| Steel Grey | `#6B7077` | Secondary geometry, support text, framing. |
| Rust | `#E2712A` | Decision node, accent line, controlled highlight. |

The secondary blue variant is retained only as a legacy bridge. Do not make blue the main DispatchTalon brand system.

## Usage Rules

- Use the primary lockup for website hero sections, sales docs, proposal covers, and broad product identification.
- Use the website header lockup in DispatchTalon website navigation.
- Use the portal login lockup on the portal login screen.
- Use the app icon for favicon, browser icon, app icon, and social avatar crop.
- Use the monochrome light mark on dark backgrounds when colour reproduction is unavailable.
- Use the monochrome dark mark only where contrast is sufficient on a light background.
- Use the blue variant only when bridging legacy LIFTIQ-era visual material.

## Minimum Size

- Digital icon minimum: `24px`.
- Print minimum: `12mm`.

For anything below those sizes, use `dispatchtalon-favicon-32.png` or the simplified app-icon family.

## Clear Space

Use the central node as the spacing reference.

Use at least one central-node height around the mark. Do not crowd the mark against copy, buttons, navigation items, ad edges, or other marks.

## Do Not Use

- Do not use the old LIFTIQ mark.
- Do not use rejected claw-only generated marks.
- Do not stretch or distort the logo.
- Do not recolour the logo outside the approved palette.
- Do not use the logo at low contrast.
- Do not crop out the central orange decision node in colour applications.
- Do not place the mark without clear space.
- Do not place positive compliance, safety, permit, legal road access, lift-engineering, or autonomous-dispatch claims near the mark.
- Do not use unofficial logo generations.

## Vector Status

The current approved source is a raster board. This pass exports locked-board PNG assets for production use and deliberately does not create raster-embedded SVG files. True SVG vectorisation should be done from designer-approved vector source or by a controlled vector recreation pass.

## Background Status

The exported PNGs preserve the locked board appearance on a dark charcoal background. Transparent-background assets should be produced later from true vector artwork or designer-approved separated source files to avoid rough edge extraction.

## Transition Note

DispatchTalon was previously developed under the LIFTIQ working name. LIFTIQ technical identifiers may remain temporarily for route, deployment, environment-variable, filename, and historical traceability stability.
