# DispatchTalon Claim-Safe Language Rules v1

Status: Internal language guardrail
Use: Sales, advisor, investor, grant, website, demo, and pilot-support copy
Owner: Pressure Systems

## Approved Positioning

DispatchTalon is a pilot-stage dispatch intelligence layer for crane, rigging, labour, plant, transport, construction, and field operations teams.

Approved core lines:

- Suitability before availability.
- The reasoning layer before the job goes out.
- DispatchTalon supports operational decisions before dispatch is confirmed.
- The dispatcher remains the decision-maker.
- DispatchTalon creates reviewable reasoning, not automatic approval.

## Approved Terms

Use:

- decision support
- dispatcher confirms
- human confirmed
- suitability before availability
- reviewable reasoning
- operational record
- assisted setup
- pilot-stage
- dispatch intelligence layer
- reasoning layer before the job goes out
- source document for pilot setup review
- human review before use
- Daily Site Log
- worker credential register
- custom credential type
- review required

## Restricted Terms

Use only with boundaries:

| Term | Safe Use |
| --- | --- |
| AI | Only if describing future experiments or bounded support. Do not imply automatic decisions. |
| import | Use "assisted setup" or "review-before-save" unless the current import path is actually live. |
| compliance | Use only to say DispatchTalon is not compliance approval or legal advice. |
| proof | Use for collected evidence, not assumptions. |
| automation | Use only for bounded workflow support, not autonomous dispatch. |
| investor-ready | Avoid unless a separate investor pack and evidence gate exist. |
| enterprise | Avoid unless discussing future requirements or risks. |

- autonomous dispatch
- compliance approved

- prevents non-compliance
- AI automatically imports

- grant eligibility

## Current Product Truth

Current live capabilities:

- Assisted Source Upload
- Daily Site Log
- worker sign in / sign out
- historical onsite lookup
- printable daily report
- custom credential types
- worker credential register
- SmartRank / decision-support logic
- Our Business setup
- personalisation profile
- diagnostic flow
- advisor demo tenants

Current boundaries:

- no automatic extraction into live records
- no autonomous dispatch
- no payroll calculation
- no compliance approval
- no safety approval
- no legal attendance certification
- no enterprise custom-role access model in production
- no deep accounting integration

## Roadmap-Only Language

Use this pattern:

"A later stage may assess [capability], but it is not live in the current pilot."

Safe examples:

- A later stage may assess review-before-save extraction, but it is not live in the current pilot.
- A later stage may assess accounting integrations, but DispatchTalon is not currently a payroll or invoicing system.
- A later stage may assess advanced role access, but current advisor demos use isolated demo tenants.

Do not present roadmap items as current product behaviour.

## Advisor / Investor-Safe Wording

Use:

- "Current pilot evidence is being collected."
- "Current research suggests..."
- "This requires source verification before external use."
- "This may align with themes worth assessing with a specialist advisor."
- "This is not a funding guarantee."
- "This is not an eligibility claim."
- "DispatchTalon supports human-confirmed allocation decisions."

Avoid:

- "proven ROI"
- "category winner"
- "no competitors"
- "fully automated"
- "compliance solved"

## Pilot-User-Safe Wording

Use:

- "Upload what you have for pilot setup review."
- "No live records are updated until confirmed."
- "Review before operational use."
- "The Daily Site Log is an operational record, not payroll approval."
- "SmartRank helps review options. The dispatcher confirms."
- "Credential status is visibility, not legal or compliance approval."

Do not pressure pilot users with inflated claims. Trust is built by clear boundaries and useful workflow evidence.
