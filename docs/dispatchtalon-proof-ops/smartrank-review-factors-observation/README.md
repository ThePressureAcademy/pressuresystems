# SmartRank Review Factors Observation Pack

Status: Internal pilot observation system
Release observed: PR #66 SmartRank Review Factors v1

## Purpose

This folder controls the first 7 days of observation for SmartRank Review Factors v1.

The goal is to learn whether placement-specific review context improves dispatch decisions without creating confusion, unsafe worker labels, or pressure to expand permissions too early.

## Files

| File | Use |
| --- | --- |
| `DISPATCHTALON_SMARTRANK_REVIEW_FACTORS_7_DAY_OBSERVATION_PLAN.md` | Daily observation plan, red flags, good-use signals, and 7-day decision gate. |
| `DISPATCHTALON_SMARTRANK_REVIEW_FACTORS_FEEDBACK_TRACKER_TEMPLATE.csv` | Tracker template for pilot feedback, usage events, confusion, and permission-change requests. |
| `DISPATCHTALON_SMARTRANK_REVIEW_FACTORS_USAGE_RULES_v1.md` | Internal rules for safe review-factor creation, language, and permission boundaries. |

## Operating Boundary

SmartRank Review Factors v1 is for placement-specific operational review only.

It ranks placement fit, not the person.

Admin users may create, edit, archive, and view Review Factors. Supervisor and dispatcher users may view relevant context only. Permission expansion is a later decision and should not happen without pilot evidence.

## External Use

Do not send this folder externally as product collateral.

This is an internal proof-ops and safety-control pack. Extracted evidence may be adapted later only after founder review, claim-safety review, and removal of private pilot information.
